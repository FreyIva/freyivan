# Аудит UX для десктопной версии — StroyControl

## Обзор

Аудит проведён для экранов ≥1024px. Оценка: навигация, формы, таблицы, визуальная иерархия, обратная связь, доступность.

---

## 1. Layout и информационная плотность

### ✅ Сильные стороны

| Элемент | Оценка |
|---------|--------|
| **max-width: 1200px** | Контент не растягивается на широких мониторах, читаемость сохранена |
| **Breadcrumbs** | Есть навигационная цепочка (Админ → Проекты → Этапы) |
| **page-header** | Заголовок + subtitle + действия в одном блоке |
| **card** | Карточки с отступами и тенью |

### 🟡 Рекомендации

| Проблема | Где | Решение |
|----------|-----|---------|
| **Широкие мониторы** | main max-width 1200px | На ≥1440px: увеличить до 1400px или добавить sidebar |
| **Дашборд админа** | 4 карточки в ряд | На 1200px — ок; на 1600px+ — добавить 5-ю колонку или увеличить размер карточек |
| **Форма создания проекта** | Много полей подряд | Группировать по смыслу (Основное / Контакты / Документы) |

## 2. Навигация

### ✅ Сильные стороны

| Элемент | Оценка |
|---------|--------|
| **Dropdown-меню** | Отчёты, Справочники, Согласование — логичная группировка |
| **Активный пункт** | `.nav-link.active` — понятно, где пользователь |
| **Бейджи** | Заявки, уведомления — видно количество |
| **Закрытие dropdown** | По клику вне, по Escape |

### 🟠 Проблемы

| Проблема | Где | Решение |
|----------|-----|---------|
| **Перегрузка меню** | Админ: 10+ пунктов + 3 dropdown | Вынести редко используемое (amoCRM, Связи ролей) в отдельный раздел или «Ещё» |
| **Дублирование** | «Отчёты» и «amoCRM» — пересечение | Объединить или явно разделить по смыслу |
| **Нет поиска** | Проекты, пользователи | Добавить поиск по названию/имени на страницах списков |
| **Нет быстрых действий** | Глобально | Рассмотреть Command Palette (Ctrl+K) для перехода по разделам |

## 3. Формы

### ✅ Сильные стороны

| Элемент | Оценка |
|---------|--------|
| **form-row** | Гибкая сетка, flex-wrap | Адаптивность |
| **Labels** | Связаны с полями (for/id) | Доступность |
| **placeholder** | Подсказки в полях | Удобство |
| **password-toggle** | Показать/скрыть пароль | Удобство |
| **phone-input** | Маска +7 (___) ___-__-__ | Валидация на лету |

### 🟠 Проблемы

| Проблема | Где | Решение |
|----------|-----|---------|
| **Длинные формы** | admin/project_edit, admin/user_edit | Разбить на шаги/вкладки или секции с заголовками |
| **Нет inline-валидации** | Все формы | Показывать ошибки до submit (required, email, телефон) |
| **Кнопка «Сохранить» внизу** | Длинные формы | Sticky footer с кнопками или дублировать вверху |
| **Select без поиска** | client_id, master_id (100+ опций) | Добавить select2/choices.js или autocomplete |
| **Нет отмены без потери** | Редактирование | «Отмена» есть; рассмотреть предупреждение при уходе с несохранёнными данными |

## 4. Таблицы и данные

### ✅ Сильные стороны

| Элемент | Оценка |
|---------|--------|
| **users-table** | Чёткая структура, zebra-строки | Читаемость |
| **hover на строках** | Подсветка при наведении | Обратная связь |
| **Ресайз колонок** | wr-list, report-cols-resizable | Гибкость |
| **Горизонтальный скролл** | users-table-wrap | На узких экранах |
| **Разные виды** | Дашборд: карточки/таблица/ленточки/сетка | Выбор пользователя |

### 🟠 Проблемы

| Проблема | Где | Решение |
|----------|-----|---------|
| **Нет сортировки** | admin/projects, users, stage_confirmations | Кликабельные заголовки для сортировки |
| **Нет пагинации** | Списки проектов, пользователей | При 50+ записях — пагинация или «Показать ещё» |
| **Длинные тексты** | Комментарии, адреса | Ограничить высоту + «Показать полностью» |
| **Нет экспорта** | stage_confirmations, edit_requests | Кнопка «Экспорт в Excel» для отчётов |
| **wr-list min-width 980px** | Отчёты работников | На десктопе ок; рассмотреть сворачиваемые колонки |

## 6. Обратная связь и взаимодействия

### ✅ Сильные стороны

| Элемент | Оценка |
|---------|--------|
| **Flash-сообщения** | success/error/warning/info | Понятная обратная связь |
| **Toast** | worker/project — «Этап подтверждён» | Не блокирует интерфейс |
| **confirm()** | Удаление, завершение проекта | Защита от случайных действий |
| **Hover на кнопках** | Изменение фона/тени | Обратная связь |
| **Переключение видов** | Дашборд, отчёты | Сохранение в localStorage |

### 🟠 Проблемы

| Проблема | Где | Решение |
|----------|-----|---------|
| **Нет индикатора загрузки** | Submit форм, синхронизация amoCRM | Spinner или disabled + «Сохранение…» |
| **Нет оптимистичного UI** | Добавление этапа, назначение | Показать изменение до ответа сервера |
| **alert() для ошибок** | worker/project — «Сначала выберите работу» | Заменить на toast или inline-сообщение |
| **Модалки** | Закрытие подэтапа, отклонение этапа | Фокус в первое поле при открытии; закрытие по Escape (частично есть) |

## 7. Доступность (a11y)

### ✅ Сильные стороны

| Элемент | Оценка |
|---------|--------|
| **focus-visible** | Кнопки, form-control | Видимый фокус при Tab |
| **aria-label** | nav-toggle, password-toggle | Для скринридеров |
| **role="group"** | display-toggle-compact | Семантика |
| **Labels** | for/id на формах | Связь полей |

### 🟠 Проблемы

| Проблема | Где | Решение |
|----------|-----|---------|
| **Нет skip-link** | Переход к main | Добавить «Перейти к контенту» для клавиатурных пользователей |
| **details/summary** | Dropdown без role="menu" | Добавить aria-expanded, aria-haspopup |
| **Модалки** | focus trap | При открытии — фокус внутри; Tab не выходит за пределы |
| **Таблицы** | Нет scope на th | Добавить scope="col" для заголовков |
| **Цветовой контраст** | Проверить subtitle, card-meta | WCAG AA: минимум 4.5:1 для текста |

## Приоритеты внедрения

### Фаза 1 (быстрые победы)

1. **Индикатор загрузки** при submit форм
2. **Skip-link** «Перейти к контенту»
3. **scope="col"** в таблицах
4. **Группировка кнопок** в page-header (основные / опасные)
5. **Предупреждение при уходе** с несохранённой формой (beforeunload)

### Фаза 2 (средний приоритет)

1. **Сортировка** в таблицах проектов, пользователей
2. **Пагинация** или «Показать ещё» для длинных списков
3. **Select с поиском** для client_id, master_id
4. **Разбивка длинных форм** на секции
5. **Toast вместо alert()** в worker/project

### Фаза 3 (расширенные улучшения)

1. **Command Palette** (Ctrl+K) для быстрой навигации
2. **Skeleton loaders** при загрузке
3. **Оптимистичный UI** для частых действий
4. **Реорганизация меню** админа (меньше пунктов в шапке)
5. **Inline-валидация** форм

## Чек-лист для проверки

- [ ] Навигация: все пункты доступны, активный выделен
- [ ] Формы: labels, placeholder, кнопки сохранения видны
- [ ] Таблицы: читаемость, hover, при необходимости — скролл
- [ ] Flash/toast: сообщения видны и понятны
- [ ] Модалки: открытие, закрытие, Escape, фокус
- [ ] Клавиатура: Tab по всем интерактивным элементам, Enter для submit
- [ ] Пустые состояния: понятный текст и CTA
- [ ] Широкий экран (1920px): нет избыточных пустых полей

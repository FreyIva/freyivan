# Аудит мобильной совместимости и UX — StroyControl

## Текущее состояние

### ✅ Что уже сделано хорошо

1. **Viewport и мета-теги** — `base.html`:
   - `viewport-fit=cover` для устройств с вырезами
   - `theme-color` для light/dark
   - `color-scheme: light dark`

2. **Навигация** — бургер-меню на ≤768px:
   - `.nav-toggle` с `aria-label`, `aria-expanded`
   - Выпадающее меню `.nav.nav-open`
   - Закрытие по клику вне меню

3. **Touch-friendly**:
   - `--touch-min: 44px` для кнопок
   - `-webkit-overflow-scrolling: touch` для скролла таблиц
   - `font-size: 16px` у `input` (предотвращает зум iOS)

4. **Адаптивные breakpoints**:
   - 360px, 480px, 600px, 768px, 860px, 900px, 1024px, 1200px
   - Формы: `form-row` → колонка на 768px
   - Карточки проектов: 1 колонка на 768px
   - Модалки: 95% ширины на 600px

5. **Таблицы** — `users-table-wrap` с `overflow-x: auto`
   - Горизонтальный скролл на узких экранах

6. **amoCRM** — есть `_amocrm_leads_mobile_list.html` (карточный вид для мобильных)

## Проблемы и рекомендации

### 🔴 Критичные

| Проблема | Где | Решение |
|----------|-----|---------|
| **`--color-dark` не определена** | `.nav` на 768px, `badge-admin`, ссылки | Добавить в `:root`: `--color-dark: #0f172a` (light), `#f8fafc` (dark) |
| **Переполнение таблицы «Закрыть день»** | `worker/project.html` — work-rows | Добавить `min-width` для `.users-table-wrap` или карточный вид на ≤600px |
| **Переполнение wr-list** | Отчёты работников — `min-width: 980px` | На ≤600px: карточный вид или `min-width: 100%` с переносом колонок |

### 🟠 Средние

| Проблема | Где | Решение |
|----------|-----|---------|
| **Переключатель тем (1–5)** | Шапка — 5 кнопок в ряд | На ≤768px: скрыть или свернуть в dropdown |
| **Impersonation select** | Шапка — длинный список | Уже `width: 100%` на 768px — ок |
| **Таблица логина** | `login-hint-table` — `min-width: 520px` | Горизонтальный скролл есть; рассмотреть карточный вид |
| **Кнопки в page-header** | Много кнопок в ряд | На 480px уже `width: 100%` — ок |
| **Фото в модалке закрытия подэтапа** | `input[type=file]` multiple | Добавить `capture="environment"` для камеры на мобильных (опционально) |

### 🟡 Улучшения UX

| Область | Рекомендация |
|---------|--------------|
| **Safe area** | Добавить `padding: env(safe-area-inset-top)` для header на iPhone X+ |
| **Pull-to-refresh** | На дашбордах (master, foreman, worker) — визуальная подсказка обновления |
| **Skeleton loaders** | При загрузке списков проектов — скелетоны вместо пустого экрана |
| **Sticky header** | На длинных страницах (отчёты, календарь) — `position: sticky` для заголовка |
| **FAB для частых действий** | «Закрыть день» у работника — плавающая кнопка внизу экрана |
| **Swipe-жесты** | В карусели фото — свайп влево/вправо |
| **Офлайн-индикатор** | Показ статуса сети при потере соединения |
| **Увеличение зоны нажатия** | Кнопки «Подтвердить этап», «Закрыть подэтап» — `min-height: 44px` |
| **Фокус и клавиатура** | При открытии модалки — фокус в первое поле; при появлении клавиатуры — прокрутка к активному полю |

## Приоритеты внедрения

### Фаза 1 (быстрые правки)
1. Добавить `--color-dark` в CSS
2. Скрыть/свернуть theme-toggle на мобильных
3. Safe-area для header
4. Увеличить touch-зоны у критичных кнопок (закрыть день, закрыть подэтап)

### Фаза 2 (таблицы и формы)
1. Карточный вид для work-rows на worker/project
2. Карточный вид для wr-list на узких экранах (или упрощённая сетка)
3. Адаптация таблицы логина (карточки для тестовых аккаунтов)

### Фаза 3 (расширенные возможности)
1. PWA / Service Worker для офлайн-доступа
2. Skeleton loaders
3. Pull-to-refresh
4. FAB для частых действий

Сценарии: кто что делает и в каком порядке

0. Модульные доступы к отчётам (RBAC)

  1. Администратор открывает карточку пользователя → «Редактировать пользователя»
  2. В блоке «Доступ к модулям» включает/выключает модули (`reports`, `amocrm`, `worker_reports`, `foreman_reports`, `tv_dashboard`)
  3. В блоке «Доступ к страницам отчётов» включает/выключает конкретные страницы
  4. После сохранения: недоступные отчёты скрываются в шапке, прямой переход по URL блокируется

1. Создание проекта

  1. Менеджер ОП (или Админ) создаёт проект → выбирает заказчика, мастера, тип строения, адрес
  2. Менеджер ОП автоматически становится ответственным (responsible_manager_id)
  3. Админ при создании может выбрать любого менеджера ОП ответственным

2. Согласование договора и сметы

  1. Менеджер ОП загружает договор и смету → уведомление Админу (или директору направления по лимиту)
  2. Админ / Директор нажимает «Одобрить» → уведомление Заказчику
  3. Заказчик подтверждает → документ согласован
  4. Если заказчик отклоняет: указывает комментарий (мин. 10 символов) → уведомление Менеджеру ОП → менеджер загружает новую версию → цепочка повторяется с шага 1

3. Модульный дом: этапы и отчёты

  1. Админ / Менеджер ОП добавляет этапы с датами начала и окончания
  2. Директор по производству назначает объект мастеру (через «Назначение объектов»)
  3. Мастер назначает объект работникам (worker_project_access)
  4. Работник закрывает день: работы + % + комментарий → Мастер или Директор по производству подтверждает %
  5. Мастер добавляет отчёт по этапу (фото + комментарий)
  6. Мастер или Директор по производству подтверждает этап → этап закрыт

4. Строительство на участке (каркас / газобетон / пенополистирол / монтаж модулей): этапы и отчёты

  1. Админ назначает Директора строительства на участке проекту (при редактировании проекта)
  2. Директор строительства на участке назначает объект прорабу (foreman_project_access)
  3. Прораб закрывает рабочий день: работы по этапу + % + комментарий
  4. Директор строительства на участке подтверждает этап → этап закрыт

5. Мастер/прораб хочет заменить фото этапа (заявка на редактирование)

  1. Мастер или Прораб создаёт заявку: «Заявки на редактирование» или кнопка на странице этапа
  2. Админ или Директор (по производству для мастера, по строительству для прораба) рассматривает заявку
  3. Одобрение → фото заменяется; отклонение → заявка закрыта

6. Мастер: вопрос по этапу, спорная ситуация

  1. Мастер заходит в проект → «Чат проекта»
  2. Пишет сообщение, при необходимости привязывает к этапу (выбор этапа в форме)
  3. В чате видят: Админ, Менеджер ОП, Директор по производству — любой может ответить
  4. Директор по производству участвует в спорных вопросах модульных домов

7. Прораб: вопрос по этапу, спорная ситуация

  1. Прораб заходит в проект → «Чат проекта»
  2. Пишет сообщение, при необходимости привязывает к этапу
  3. В чате видят: Админ, Менеджер ОП, Директор строительства на участке — любой может ответить

8. Взятие проекта (делегирование)

  1. Ответственный ОП неактивен > 7 дней (не редактирует проект, этапы, не загружает документы)
  2. Другой Менеджер ОП → «Взять проект» → «Запросить»
  3. Админ → «Заявки на проекты» → одобряет или отклоняет
  4. При одобрении: проект переходит заявителю, оба получают уведомление
  5. Апелляция: бывший ответственный в течение 24 часов может нажать «Оспорить» → заявка на апелляцию с причиной → Админ в «Заявки на проекты» (вкладка Апелляции) может отменить передачу или отклонить апелляцию. Решение окончательное.

9. Завершение проекта

  1. Админ заходит в этапы проекта → кнопка «Завершить проект»
  2. Проект исчезает с дашборда, остаётся в списке проектов с пометкой «Завершён»
  3. Кнопка «Открыть проект» — вернуть проект на дашборд

10. Отчёты amoCRM для отдела продаж

  1. Админ назначает пользователю модуль и страницы amoCRM-отчётов
  2. Менеджер ОП / РОП / Маркетолог открывают доступные отчёты: «Потенциальные клиенты», «Проекты Sherwood Home», «Источники лидов», TV-дашборд
  3. Данные подтягиваются из amoCRM и обновляются автоматически (через кэш синхронизации)
  4. Права действуют и в навигации, и при прямом переходе по URL
